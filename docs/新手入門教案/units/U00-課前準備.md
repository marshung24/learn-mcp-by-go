## ③ 核心觀念

### 1. Go 環境架構

```
$HOME/
├── go/                    # GOPATH（預設）
│   ├── bin/              # 編譯後的執行檔
│   ├── pkg/              # 快取的套件
│   └── src/              # 原始碼（舊式，現已少用）
└── your-project/         # 專案目錄（任意位置）
    ├── go.mod            # 模組定義
    ├── go.sum            # 依賴鎖定
    └── main.go           # 程式碼
```

### 2. Go Modules（go.mod）

- Go 1.11+ 引入的依賴管理機制
- 每個專案都有獨立的 `go.mod` 檔案
- 執行 `go mod init <module-name>` 初始化

### 3. 環境變數

| 變數 | 說明 | 查看指令 |
|------|------|---------|
| `GOROOT` | Go 安裝路徑 | `go env GOROOT` |
| `GOPATH` | 工作區路徑 | `go env GOPATH` |
| `GOPROXY` | 套件下載代理 | `go env GOPROXY` |

### 4. Claude Desktop 設定檔位置

| 系統 | 路徑 |
|------|------|
| macOS | `~/Library/Application Support/Claude/claude_desktop_config.json` |
| Windows | `%AppData%\Claude\claude_desktop_config.json` |

### 5. Claude Code 設定檔位置

| 範圍 | 路徑 | 說明 |
|------|------|------|
| 專案層級 | `.mcp.json`（專案根目錄） | 僅對該專案生效 |
| 使用者層級 | `~/.claude/settings.json` | 對所有專案生效 |

## ④ 動手做

### [必做] 4.1 安裝 Go

**macOS (Homebrew)**
```bash
brew install go
```

**macOS/Linux (官方安裝包)**
```bash
# 下載後執行
sudo tar -C /usr/local -xzf go1.21.x.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin
```

**Windows**
- 下載 https://go.dev/dl/ 的 .msi 安裝檔
- 執行安裝，完成後重開終端

### [必做] 4.2 驗證安裝

```bash
# 確認版本
go version
# 預期：go version go1.21.x darwin/arm64 (或類似)

# 確認環境變數
go env GOROOT
go env GOPATH

# 建立測試專案
mkdir -p ~/mcp-test && cd ~/mcp-test
go mod init mcp-test
```

### [必做] 4.3 建立 Hello World

建立 `main.go`：

```go
package main

import "fmt"

func main() {
    fmt.Println("Go is ready for MCP!")
}
```

執行：

```bash
go run main.go
# 預期輸出：Go is ready for MCP!
```

### [必做] 4.4 安裝 MCP Go SDK

```bash
go get github.com/modelcontextprotocol/go-sdk/mcp
```

確認安裝：

```bash
cat go.mod
# 應看到 require github.com/modelcontextprotocol/go-sdk vX.X.X
```

### [必做] 4.5 安裝 VS Code 與 Go 擴充

1. 下載安裝 VS Code
2. 開啟 VS Code
3. 按 `Cmd+Shift+X` (macOS) 或 `Ctrl+Shift+X` (Windows)
4. 搜尋 "Go" 並安裝官方擴充
5. 開啟任意 `.go` 檔案，VS Code 會提示安裝 gopls 等工具，點選 Install All

### [延伸] 4.6 安裝 Claude Desktop

1. 前往 https://claude.ai/download
2. 下載並安裝 Claude Desktop
3. 登入 Claude 帳號

### [延伸] 4.7 確認 Claude Desktop 設定檔位置

**macOS**
```bash
# 確認目錄存在
ls -la ~/Library/Application\ Support/Claude/

# 如果不存在，先開啟一次 Claude Desktop 讓它建立
```

**Windows (PowerShell)**
```powershell
ls $env:AppData\Claude\
```

### [延伸] 4.8 安裝 Node.js（用於 MCP Inspector）

MCP Inspector 需要 Node.js 18+ 環境：

**macOS (Homebrew)**
```bash
brew install node
```

**其他平台**
前往 https://nodejs.org/ 下載安裝

驗證安裝：
```bash
node --version  # 需顯示 v18.x 或更高
npm --version
```

### [延伸] 4.9 安裝 Claude Code

**macOS**
```bash
brew install claude-code
```

**npm（跨平台）**
```bash
npm install -g @anthropic-ai/claude-code
```

安裝完成後驗證：
```bash
claude --version
```

### [延伸] 4.10 確認 Claude Code 設定

```bash
# 查看目前的 MCP Server 設定
claude mcp list

# 測試新增 MCP Server（後續單元會用到）
# claude mcp add <name> -- <command>
```

## ⑤ 踩坑提示

### 踩坑 1：go: command not found

**現象**
```bash
$ go version
zsh: command not found: go
```

**原因**
Go 未加入 PATH 環境變數。

**解法**
```bash
# macOS/Linux：編輯 ~/.zshrc 或 ~/.bashrc
export PATH=$PATH:/usr/local/go/bin
export PATH=$PATH:$(go env GOPATH)/bin

# 重新載入
source ~/.zshrc
```

---

### 踩坑 2：go mod init 失敗

**現象**
```bash
$ go mod init
go: cannot determine module path for source directory
```

**原因**
未指定模組名稱。

**解法**
```bash
go mod init my-module-name
# 或使用專案路徑
go mod init github.com/username/project
```

---

### 踩坑 3：VS Code 無法找到 Go

**現象**
VS Code 開啟 .go 檔案時，顯示 "go: command not found" 或無法啟動 gopls。

**原因**
VS Code 未讀取到 shell 的 PATH 設定。

**解法**
1. 從終端機啟動 VS Code：
   ```bash
   code .
   ```
2. 或在 VS Code 設定中指定 Go 路徑：
   - 開啟設定 (Cmd+,)
   - 搜尋 "Go: GOROOT"
   - 設定為 `/usr/local/go`
