# 結構示範檔

> 用途：MCP Server 通用程式骨架速查。
> 原則：以 pattern 方式呈現可重用的結構，各單元教案與 examples/ 對照本檔理解架構。
> 教案省略處的參照優先順序：結構示範檔（通用 pattern）→ examples/（完整實作）→ 先前單元（特定案例）。

---

## 1. 最小 Stdio MCP Server 骨架

適用情境：任何 Stdio 模式的 MCP Server，聚焦在 capability 註冊點。

```go
package main

import (
	"context"
	"log"
	"os"

	"github.com/modelcontextprotocol/go-sdk/mcp"
)

func main() {
	// Stdio 模式下，stderr 才能安全輸出 log
	log.SetOutput(os.Stderr)

	server := mcp.NewServer(&mcp.Implementation{
		Name:    "demo-server",
		Version: "1.0.0",
	}, nil)

	// 在這裡註冊本單元新增的能力
	// mcp.AddTool(server, ...)
	// server.AddResource(...)
	// server.AddPrompt(...)

	if err := server.Run(context.Background(), &mcp.StdioTransport{}); err != nil {
		log.Fatal(err)
	}
}
```

**教案內引用寫法**：

```go
func main() {
	// Server 啟動骨架（見結構示範檔 §1）log.SetOutput ... (略)

	// ===== 以下為本單元新增內容 =====

	// 註冊 greet Tool
	mcp.AddTool(server, ...)
}
```

> 完整實作見 `examples/U03-tools/main.go`。

## 2. HTTP MCP Server 骨架

適用情境：任何 HTTP 模式的 MCP Server，聚焦在 route、middleware、auth 等差異。

```go
package main

import (
	"log"
	"net/http"
	"os"

	"github.com/modelcontextprotocol/go-sdk/mcp"
)

func main() {
	log.SetOutput(os.Stderr)

	server := mcp.NewServer(&mcp.Implementation{
		Name:    "http-demo",
		Version: "1.0.0",
	}, nil)

	mcpHandler := mcp.NewStreamableHTTPHandler(func(req *http.Request) *mcp.Server {
		return server
	}, nil)

	// 在這裡套用本單元的 route / middleware / auth
	http.Handle("/mcp", mcpHandler)

	if err := http.ListenAndServe(":8080", nil); err != nil {
		log.Fatal(err)
	}
}
```

## 3. Tool Handler 骨架

適用情境：任何 Tool 的 handler 實作，聚焦在輸入結構與回傳邏輯的差異。

```go
// ExampleInput 定義 Tool 的輸入參數
type ExampleInput struct {
	Field string `json:"field" jsonschema:"欄位說明"`
}

// exampleHandler 處理 Tool 的呼叫請求
func exampleHandler(ctx context.Context, req *mcp.CallToolRequest, input ExampleInput) (
	*mcp.CallToolResult, any, error,
) {
	// 這裡處理本單元邏輯
	result := "..."

	return &mcp.CallToolResult{
		Content: []mcp.Content{
			&mcp.TextContent{Text: result},
		},
	}, nil, nil
}
```

**註冊呼叫**（三步驟的第三步）：

```go
mcp.AddTool(server, mcp.Tool{
	Name:        "example",
	Description: "Tool 說明",
}, exampleHandler)
```

**教案內引用寫法**：

```go
// Tool handler 骨架（見結構示範檔 §3）
// 本節聚焦 CalculateInput 欄位與 switch input.Operator 的差異邏輯
```

> 完整實作見對應單元的 `examples/` 目錄。

## 4. Resource Handler 骨架

```go
// exampleResourceHandler 處理 Resource 的讀取請求
func exampleResourceHandler(ctx context.Context, req *mcp.ReadResourceRequest) (
	*mcp.ReadResourceResult, error,
) {
	content := "..."

	return &mcp.ReadResourceResult{
		Contents: []*mcp.ResourceContents{
			{
				URI:      "file://example",
				MIMEType: "text/plain",
				Text:     content,
			},
		},
	}, nil
}
```

**註冊呼叫**：

```go
server.AddResource(&mcp.Resource{
	URI:  "file://example",
	Name: "Example Resource",
}, exampleResourceHandler)
```

## 5. Prompt Handler 骨架

```go
// examplePromptHandler 處理 Prompt 的取得請求
func examplePromptHandler(ctx context.Context, req *mcp.GetPromptRequest) (
	*mcp.GetPromptResult, error,
) {
	promptText := "..."

	return &mcp.GetPromptResult{
		Description: "Prompt 說明",
		Messages: []*mcp.PromptMessage{
			{
				Role:    "user",
				Content: &mcp.TextContent{Text: promptText},
			},
		},
	}, nil
}
```

**註冊呼叫**：

```go
server.AddPrompt(&mcp.Prompt{
	Name:        "example-prompt",
	Description: "Prompt 說明",
}, examplePromptHandler)
```

## 使用邊界

| 適合抽到本檔 | 不適合抽到本檔 |
|-------------|--------------|
| server 啟動骨架 | 第一次出現的完整核心教學範例 |
| handler 簽名 | 該章唯一的新概念 |
| 固定回傳結構 | 需要逐行講解的協定細節 |
| 重複的註冊模式 | 容易因抽離而失去上下文的錯誤處理流程 |
